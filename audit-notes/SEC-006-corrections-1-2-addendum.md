## CORRECTION 1 — remember_token Reclassification

### Scraper Login Flow Verification

**Question:** Does the scraper authenticate with owner credentials before fetching the pages that produced the debug HTML files?

**File reference:** `scripts/debug_rg_pitchers.mjs`, function `login()` (lines 21–56); `server/rotogrinderProxy.ts`, Layer 3 fresh login block (lines 508–582).

**Evidence:**

`scripts/debug_rg_pitchers.mjs` `login()` function:
```
const USERNAME = process.env.ROTOGRINDERS_USERNAME;
const PASSWORD = process.env.ROTOGRINDERS_PASSWORD;
...
loginRes = await fetch(`${RG_BASE}/sign-in`, {
  method: "POST",
  body: new URLSearchParams({ username: USERNAME, password: PASSWORD }).toString(),
  ...
});
```

`server/rotogrinderProxy.ts` Layer 3 login block:
```
const username = process.env.ROTOGRINDERS_USERNAME;
const password = process.env.ROTOGRINDERS_PASSWORD;
...
loginRes = await fetch(`${RG_BASE}/sign-in`, { method: "POST", body: ... });
```

**VERIFIED: YES** — both the `debug_rg_pitchers.mjs` script and `rotogrinderProxy.ts` authenticate via POST to `https://rotogrinders.com/sign-in` using `ROTOGRINDERS_USERNAME` and `ROTOGRINDERS_PASSWORD` before fetching any page. The debug HTML files were produced by an authenticated session. **VERIFIED**

**VERIFIED:** `ROTOGRINDERS_USERNAME` prefix in process env is `Lucian` — this is the `lucianobets` account (confirmed against `ALLOWED_USERNAMES = new Set(["prez", "lucianobets", "sippi"])` in `rotogrinderProxy.ts` line 43). **VERIFIED**

**Page body context of remember_token:**

```
"twitter_username":"LucianoMLB","slack_username":null,"notes":null,
"alerts":1,"payment_token":"cus_PoMNPjA4s0sJBO","fingerprint":null,
"remember_token":"[REDACTED]","device_token":null,...
```

The `remember_token` appears inside a JSON blob embedded in the page body, within a user object that includes `twitter_username: "LucianoMLB"`. This is the authenticated user's own session data returned by RotoGrinders after login. **VERIFIED**

### Reclassification

The prior label **THIRD-PARTY** is **INCORRECT**. The `remember_token` is the persistent session token for the `lucianobets` RotoGrinders account — an owner-controlled credential used by the scraper. It is present in the page body because RotoGrinders embeds the authenticated user's account data in the rendered HTML.

**Corrected label: OWNER SESSION CREDENTIAL (exposed, private-repo scope)**

**Remediation already planned per user directive:** RotoGrinders password change and session invalidation for the `lucianobets` account will invalidate this token. **VERIFIED (remediation path)**
